HTML semántico: el esqueleto
que nadie ve pero todos necesitan

Aprender a escribir HTML correcto no es solo saber las etiquetas — es entender el significado que le das a cada parte de tu página. En este artículo exploramos el HTML semántico desde cero.

HTML

¿Qué es el HTML semántico?

Cuando empezamos a aprender HTML, la tentación es usar <div> para todo. Y funciona… visualmente. Pero HTML no es solo diseño: es lenguaje de marcado, y cada etiqueta tiene un significado. Ese significado es lo que llamamos semántica.

El HTML semántico consiste en elegir la etiqueta que mejor describe el contenido que contiene, no la que se ve mejor por defecto.

"Un documento bien estructurado con HTML semántico le dice al navegador, a los lectores de pantalla y a los motores de búsqueda qué es cada cosa, no solo dónde está." — MDN Web Docs, Introducción a HTML

¿Por qué importa la semántica?

Usar HTML semántico tiene tres beneficios principales que van mucho más allá de "el código se ve más ordenado":

  1. Accesibilidad: Los lectores de pantalla usan las etiquetas semánticas para navegar. Un <nav> les indica que ahí está la navegación; un <main>, el contenido principal.
  2. SEO: Google y otros buscadores analizan la estructura semántica para entender de qué trata tu página. Un <h1> tiene más peso que un <div class="titulo">.
  3. Mantenibilidad: Un equipo de desarrolladores puede entender la estructura de una página más rápido si el HTML es semántico. El código se documenta a sí mismo.

Estructura semántica básica de una página

Toda página web tiene áreas bien definidas. HTML5 introdujo etiquetas específicas para cada una. Compara estos dos ejemplos:

HTML — Sin semántica ❌ 
<div id="header">
  <div id="nav">...</div>
</div>
<div id="main">
  <div class="article">...</div>
  <div id="sidebar">...</div>
</div>
<div id="footer">...</div>
HTML — Con semántica ✅ 
<header>
  <nav>...</nav>
</header>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>

El resultado visual puede ser idéntico, pero el segundo ejemplo transmite significado: un navegador, un robot y un lector de pantalla entienden exactamente qué hay en cada sección.

Las etiquetas semánticas más importantes

Estas son las etiquetas semánticas de HTML5 que debes conocer y usar en cada proyecto:

  • <header> — Encabezado de la página o de una sección. Suele contener el logo, el título y la navegación principal.
  • <nav> — Bloque de enlaces de navegación. Solo se usa para conjuntos de enlaces importantes (menú principal, paginación, breadcrumbs).
  • <main> — Contenido principal y único de la página. Solo debe haber un <main> por documento.
  • <article> — Contenido independiente y auto-contenido: un post de blog, una noticia, un comentario, un widget.
  • <section> — Agrupación temática de contenido con su propio encabezado. No es un reemplazo de <div>.
  • <aside> — Contenido relacionado pero separado del flujo principal: sidebar, anuncios, notas al margen.
  • <footer> — Pie de página o de sección. Contiene metadatos, derechos de autor, enlaces secundarios.
  • <figure> y <figcaption> — Para imágenes, diagramas o bloques de código con su descripción.
  • <time> — Para fechas y horas legibles por máquinas. Usa el atributo datetime para el valor formal.
  • <mark> — Texto resaltado por relevancia en un contexto, como resultados de búsqueda.

La jerarquía de encabezados

Los encabezados <h1> al <h6> forman una estructura de árbol. Usa <h1> para el título principal de la página (solo uno), <h2> para las secciones principales, <h3> para subsecciones, y así sucesivamente.

Ejemplo práctico completo

Veamos cómo estructurar un artículo de blog usando HTML semántico correcto:

HTML 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mi primer artículo</title>
</head>
<body>

  <header>
    <a href="/" aria-label="Ir al inicio">
      <img src="logo.svg" alt="Logo del blog">
    </a>
    <nav aria-label="Navegación principal">
      <ul>
        <li><a href="/articulos">Artículos</a></li>
        <li><a href="/acerca">Acerca de</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h1>HTML semántico explicado</h1>
        <p>Por <address>Ana Martínez</address></p>
        <time datetime="2025-04-15">15 de abril, 2025</time>
      </header>

      <p>Contenido del artículo...</p>

      <figure>
        <img src="diagrama.png" alt="Diagrama de estructura HTML semántica">
        <figcaption>Estructura semántica de una página web</figcaption>
      </figure>

      <footer>
        <p>Etiquetas: <a href="/tags/html">HTML</a></p>
      </footer>
    </article>

    <aside aria-label="Artículos relacionados">
      <h2>También te puede interesar</h2>
      <!-- Lista de artículos relacionados -->
    </aside>
  </main>

  <footer>
    <p><small>© 2025 DevBlog IDS</small></p>
  </footer>

</body>
</html>

Consejos finales

  • Siempre usa lang en <html>. Indica el idioma del documento a los lectores de pantalla y traductores.
  • No saltes niveles de encabezados. De <h1> a <h3> sin un <h2> es un error de accesibilidad.
  • Agrega aria-label a elementos repetidos. Si tienes varios <nav>, diferéncialos con etiquetas descriptivas.
  • El alt en imágenes no es opcional. Si la imagen es decorativa, usa alt="" vacío. Si comunica información, descríbela con precisión.
  • Valida tu HTML. Usa el W3C Validator para verificar que tu documento no tiene errores semánticos.
